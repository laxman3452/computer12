1.5 Database Models

Class 12 Computer Science - DBMS

📌 Introduction

A database model defines the logical structure of a database and determines how data is stored, organized, and manipulated in DBMS.

🌳 1. Hierarchical Model

Data is organized in a tree-like structure where each parent can have multiple children, but a child has only one parent.

📌 Features:

  • One-to-many relationship
  • Tree structure (root → branches)
  • Fast navigation for structured data

📊 Diagram:

        University
            |
    ----------------
    |              |
  CS Dept       IT Dept
    |              |
 Students       Students

🕸️ 2. Network Model

Data is represented in a graph structure where multiple relationships are possible.

📌 Features:

  • Many-to-many relationship
  • Flexible structure
  • Complex data handling

📊 Diagram:

  Student ----- Course
     \           /
      \         /
       Teacher

📋 3. Relational Model

Data is stored in tables (relations) consisting of rows and columns. It is the most widely used database model today.

📌 Features:

  • Table-based structure
  • Uses SQL language
  • Easy to manage and understand

📊 Example Table:

+----+--------+-----+
| ID | Name   | Age |
+----+--------+-----+
| 1  | Ram    | 18  |
| 2  | Sita   | 19  |
+----+--------+-----+

⚖️ Comparison of Database Models

Model Structure Relationship Use
Hierarchical Tree One-to-many Old systems
Network Graph Many-to-many Complex systems
Relational Table All types Modern DBMS (MySQL)
