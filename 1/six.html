📚 Centralized vs Distributed Database

Class 12 DBMS - Easy & Visual Learning

🧠 Introduction

A database system is used to store and manage data. It can be stored in one place or in multiple places.

🏢 1. Centralized Database

Definition: A centralized database is a system where all data is stored in a single central location (server).

✔ Features

  • All data in one server
  • Easy to manage
  • Low cost

❌ Disadvantages

  • If server fails → whole system stops
  • Slow for remote users

📌 Diagram

        Users
       /  |  \
      /   |   \
     ▼    ▼    ▼
   [ CENTRAL SERVER ]
        DATABASE

Example: A single school database stored in one computer server.

🌐 2. Distributed Database

Definition: A distributed database stores data in multiple locations but connects them using a network.

✔ Features

  • Data stored in multiple servers
  • Faster access
  • More reliable

❌ Disadvantages

  • Complex management
  • High cost

📌 Diagram

  Branch A        Branch B        Branch C
   Server          Server          Server
     |               |               |
     ▼               ▼               ▼
  ---------- NETWORK CONNECTED DATABASE ----------

Example: Bank branches storing data in different cities but connected together.

⚖️ Comparison Table

Feature Centralized Distributed
Location Single server Multiple servers
Speed Slower for remote users Faster access
Reliability Low High
Management Easy Complex

🌍 Real-Life Examples

  • Centralized: Small school database
  • Distributed: Banking system (different branches)
  • Distributed: Google services (multiple servers worldwide)
