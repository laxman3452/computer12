1.2 DBMS Concepts: Field, Record, Object & Keys

📌 Field (Smallest Unit)

A field is the smallest unit of data in a database. It represents a single attribute.

Example: Name, Age, Address, Roll No

📌 Record

A record is a collection of related fields representing one entity instance.

Example: (Ram, 20, Kathmandu)

📌 Object (DBMS Context)

An object is a real-world entity represented in the database.

Example: Student, Teacher, Book, Employee

🔑 Keys in DBMS

  • Primary Key: Unique identifier of a record
  • Candidate Key: All possible unique keys
  • Alternate Key: Candidate keys not selected as primary

📊 Example (Student Table)

StudentID Name Email Phone
101 Ram ram@gmail.com 9800000000

✔ Primary Key = StudentID (unique)
✔ Candidate Keys = StudentID, Email, Phone
✔ Alternate Keys = Email, Phone
